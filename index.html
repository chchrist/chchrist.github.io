Connection: keep-alive and performance

  • Published: 1st June 2020
  • Updated: 1st June 2020

When you need to make multiple requests to an API it is better to send the Connection: keep-alive header to reduce latency in subsequent requests.

More about that here: https://en.wikipedia.org/wiki/HTTP_persistent_connection
